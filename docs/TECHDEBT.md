**Invariant #9 (new, from TD-2) — «built but unreachable = dead integration».** Every curated engine capability MUST have a registered CLI subparser-choice AND be exercised end-to-end. **Acceptance criterion = reachable e2e, not only a green unit test.** Enforced by `test_phase15_reachability.py` (fails on any orphan). Added to the project root-cause invariant table (AGENT.MD).

**Invariant #10 (new, from TD-4) — «shipped ≠ valuable».** A curated capability that moves *no* measured metric — neither a benchmark score (DRACO) nor a trust metric (`bench/trust/`: determinism, unsupported-claim suppression recall+precision, citation completeness, checkpoint fidelity) — is a cut candidate, not a keep. This is invariant #9 applied to *value* rather than *reachability*. **Acceptance for a simplification cut = the trust scorecard is byte-identical before/after** (the cut removed dead code, not value). The reverse tension is real: a feature can be *reachable* (inv #9 satisfied) yet *valueless* (inv #10 violated) — e.g. `plan.py` / the feedback-ledger, deferred in v1.5 precisely because cutting a reachable capability is a feature decision, not a cleanup.
